# devops-netology

Благодаря добавленному файлу `.gitignore` в каталоге `terraform` будут игнорироваться следующие типы файлов:

- Все файлы и директории с именем `.terraform` (включая их содержимое)
- Все файлы, содержащие в названии `.tfstate` (включая файлы с дополнительными суффиксами)
- Файлы логов аварийных завершений с именами `crash.log` или `crash.*.log`
- Все файлы с расширением `.tfvars` или `.tfvars.json` (файлы переменных)
- Файлы переопределений конфигурации с именами `override.tf`, `override.tf.json`, `*_override.tf`, `*_override.tf.json`
- Файлы блокировок состояний с именем `.terraform.tfstate.lock.info`
- Конфигурационные файлы CLI Terraform с именами `.terraformrc` или `terraform.rc`

Примечание: Основные файлы конфигурации Terraform с расширением `.tf` НЕ игнорируются.

















1. .terraform/ - локальные директории Terraform с плагинами и кэшем
2. *.tfstate - все файлы состояний Terraform
3. *.tfstate.* - резервные копии и временные файлы состояний
4. crash.log - файлы логов аварийных завершений
5. crash.*.log - все файлы логов аварийных завершений
6. *.tfvars - файлы переменных (они содержат чувствительные данные)
7. *.tfvars.json - JSON-файлы переменныe
8. override.tf - файл переопределений
9. override.tf.json - файл переопределений
10. *_override.tf - файл переопределений
11. *_override.tf.json - файл переопределений
12. .terraform.tfstate.lock.info - файлы блокировок состояний
13. .terraformrc - конфигурационные файлы CLI
14. terraform.rc - конфигурационные файлы CLI

.tf - НЕ игнорируются это основные файлы конфигов
